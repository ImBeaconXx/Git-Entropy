# Git-Entropy (开发中)
> 拒绝盲目重构。为现代团队打造的数据驱动型技术债分析工具。

[English Version](./README.md)

### 📊 痛点
传统的静态代码分析工具（如 SonarQube）会告诉你代码写得「好不好」，但不会告诉你它「痛不痛」。

在实际开发中：
- 一段复杂的函数如果三年没动过，它不是技术债，它只是古董。
- 一段简单的函数如果每个月都要修改 50 次，那它就是团队的效率杀手。

**Git-Entropy** 结合了 **代码变动频率 (Churn)** 与 **结构复杂度 (Complexity)**，精确量化代码库中的「熵值」，帮你找出最值得重构的地方。

### 🧠 核心逻辑
我们引入了「时间衰减模型」来计算变动频率，确保最近的提交（Commit）比三年前的提交拥有更高的权重。

- **Tree-sitter 解析：** 不同于简单的正则匹配，我们通过 AST（抽象语法树）精确计算循环复杂度。
- **本地优先：** 100% 本地运行，无需担心代码隐私问题。
- **性能至上：** 基于 Rust 构建，即使是拥有数万次提交的大型仓库，也能在秒级完成扫描。

### 🛠️ 技术架构
- **核心：** Rust (利用 `git2-rs`)，保证极速的日志遍历。
- **解析引擎：** 基于 Tree-sitter 的多语言支持。
- **界面：** 极简的命令行界面 (TUI)，并支持导出交互式的热力图 (Heatmap)。
